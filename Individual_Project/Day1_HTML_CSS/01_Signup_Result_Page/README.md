# Sign Up Page

수업 시간에 실습한 회원가입 Form을 바탕으로 만든 회원가입 및 결과 페이지입니다. 사용자가 회원 정보를 입력하고 회원가입 버튼을 누르면 입력 조건을 확인한 뒤 결과 페이지로 이동하며, 전달된 값은 표로 확인할 수 있습니다.

기본 과제 기능 외에 관심 분야 확장, 결과 페이지 정리, 생년월일 날짜 제한, 회원가입 도우미를 추가했습니다.

## 실행 방법

프로젝트 폴더에서 아래 파일을 브라우저로 실행합니다.

`signup.html`

## 주요 구성

- 회원가입 정보 입력 Form
- 기본 회원 정보, 선택 정보, 약관 동의 구분
- 입력값 유효성 검사
- 회원가입 결과 페이지
- 관심 분야 복수 선택
- 회원가입 입력 도우미
- 다시 입력하기 기능
- 화면 크기에 따른 반응형 배치

## 과제 요구사항 적용

- `text`, `password`, `email`, `tel`, `date` 입력 사용
- 성별은 `radio`, 관심 분야는 `checkbox`로 구성
- 지역은 `select`, 자기소개는 `textarea` 사용
- `fieldset`과 `legend`를 이용해 입력 항목 구분
- 이름, 아이디, 비밀번호, 이메일, 약관 동의에 `required` 적용
- 아이디는 영문과 숫자 4~12자로 제한
- 비밀번호는 8자 이상으로 제한
- 전화번호는 `010-0000-0000` 형식으로 입력
- 상단 헤더에 `position: fixed`와 `z-index` 적용
- 입력창의 `focus`와 버튼의 `hover` 효과 적용
- GET 방식으로 입력값을 `result.html`에 전달
- 전달된 값을 결과 페이지의 `table`에 표시
- 관심 분야 복수 선택값을 `getAll()`로 처리
- 미디어쿼리를 이용한 반응형 레이아웃 적용

## 창의적 확장 부분

수업에서 배운 Form 요소와 유효성 검사 속성을 실제 회원가입 화면에 맞게 구성했습니다. 입력 항목은 기본 회원 정보, 선택 정보, 약관 동의로 나누었고, 필수 입력값과 선택 입력값이 구분되도록 만들었습니다.

결과 페이지에서는 `URLSearchParams`를 사용해 주소로 전달된 값을 가져왔습니다. 일반 입력값은 `get()`으로 가져오고, 여러 개를 선택할 수 있는 관심 분야는 `getAll()`을 사용해 전부 표시했습니다.

생년월일은 페이지를 실행한 날짜 이후를 선택할 수 없도록 JavaScript에서 현재 날짜를 구해 `max` 값에 적용했습니다.

### 회원가입 도우미 챗봇

회원가입 화면 오른쪽에 입력 도우미를 추가했습니다. 아이디, 비밀번호, 이메일, 전화번호, 필수 항목, 관심 분야, 약관 동의에 관한 질문을 입력하면 관련된 안내를 보여 줍니다.

실제 AI 서비스나 외부 API를 사용한 것은 아니며, 질문에 포함된 단어를 확인한 뒤 미리 작성해 둔 답변을 보여 주는 방식입니다.

### 관심 분야 확장

관심 분야를 기본 항목에서 AI, 데이터, 개발 관련 분야 등 총 20개로 늘렸습니다. 여러 항목을 선택하면 결과 페이지에 선택한 내용이 모두 표시됩니다.

### 결과 페이지 개선

결과표를 기본 회원 정보, 선택 정보, 동의 정보로 나누어 확인하기 쉽게 만들었습니다. 입력하지 않은 항목은 별도 문구로 표시하고, 비밀번호는 실제 문자 대신 입력한 길이만큼 점으로 표시했습니다.

### UI 변경

전체 배경은 흰색과 연한 회색을 사용하고, 빨간색과 주황색을 포인트 색상으로 적용했습니다. SKALA 로고를 헤더에 배치하고 회원가입 Form과 결과표는 카드 형태로 구성했습니다.
