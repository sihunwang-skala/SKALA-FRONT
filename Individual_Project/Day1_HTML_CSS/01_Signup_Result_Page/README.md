# Sign Up Page

> 수업 시간에 실습한 HTML Form을 바탕으로 만든 회원가입 및 결과 페이지입니다.  
> `signup.html`회원 정보를 입력하면 입력 조건을 확인한 뒤 `result.html`로 이동하고, 전달된 값은 표에서 다시 확인할 수 있습니다.

---

## 실행 방법

프로젝트 폴더에서 아래 파일을 브라우저로 실행합니다.

```text
signup.html
```

GitHub 저장소:

```text
https://github.com/sihunwang-skala/SKALA-FRONT
```

## 주요 기능

- 회원가입 정보 입력 Form
- 기본 회원 정보, 선택 정보, 약관 동의 구분
- 필수값과 입력 형식 확인
- GET 방식으로 결과 페이지 이동
- 입력한 회원 정보를 표로 출력
- 관심 분야 복수 선택 및 전체 출력
- 생년월일 미래 날짜 선택 제한
- 회원가입 입력 도우미
- 화면 크기에 따른 반응형 배치

## 과제 요구사항 확인

과제 안내에 나온 항목을 기준으로 최종 코드에 적용했는지 확인했습니다.

| 구분 | 요구사항 | 적용 내용 | 확인 |
|---|---|---|:---:|
| HTML Form | `name`, `placeholder`, `value` | 전송 이름, 안내 문구, 선택값을 각 입력 요소에 설정 | ✅ |
| 입력 유형 | `text`, `password`, `email`, `tel`, `date` | 기본 회원 정보 입력에 각 유형 사용 | ✅ |
| 선택 요소 | `radio`, `checkbox`, `select`, `textarea` | 성별, 관심 분야, 지역, 자기소개에 적용 | ✅ |
| 입력 그룹 | `fieldset`, `legend` | 기본 정보, 선택 정보, 약관 동의를 구분 | ✅ |
| 필수 입력 | `required` | 이름, 아이디, 비밀번호, 이메일, 약관 동의에 적용 | ✅ |
| 아이디 검사 | 영문·숫자 4~12자 | `pattern="[A-Za-z0-9]{4,12}"` 적용 | ✅ |
| 비밀번호 검사 | 8자 이상 | `minlength="8"` 적용 | ✅ |
| 전화번호 검사 | `010-0000-0000` | `tel`과 전화번호 `pattern` 적용 | ✅ |
| 길이·날짜 제한 | `maxlength`, `min`, `max` | 문자 길이와 생년월일 범위 제한 | ✅ |
| CSS 기본 | Box Model, `box-sizing` | `margin`, `padding`, `border`, `border-box` 적용 | ✅ |
| 고정 헤더 | `position: fixed`, `z-index` | 스크롤 중에도 헤더가 상단에 유지 | ✅ |
| 상태 스타일 | `:focus`, `:hover`, 특이도 | 입력 포커스와 버튼·선택 항목 효과 적용 | ✅ |
| 페이지 이동 | GET → `result.html` | `form`의 `action`, `method`로 값 전달 | ✅ |
| 결과 출력 | `table` 사용 | 전달된 회원 정보를 항목별 표로 표시 | ✅ |
| 복수값 처리 | `URLSearchParams.getAll()` | 관심 분야의 여러 선택값을 모두 출력 | ✅ |
| 반응형 화면 | 미디어쿼리 | 화면 너비에 따라 폼과 도우미 배치 변경 | ✅ |

## 수업 내용을 바탕으로 확장한 부분

### 생년월일 제한

생년월일의 최대 날짜를 HTML에 고정하지 않고 `validation.js`에서 현재 날짜를 구해 `max` 값으로 넣었습니다. 그래서 오늘 이후 날짜는 선택할 수 없습니다.

### 관심 분야 확장

관심 분야를 프론트엔드, 백엔드, AI Agent, 머신러닝, 딥러닝, 데이터 분석, MLOps 등 총 20개로 늘렸습니다. 체크박스의 `name`을 모두 `interest`로 맞추고, 결과 페이지에서 `getAll()`로 선택한 값을 전부 가져오도록 했습니다.

### 결과 페이지 정리

결과표를 기본 회원 정보, 선택 정보, 동의 정보로 나눴습니다. 선택하지 않은 항목은 별도 문구로 표시하고, 비밀번호는 실제 문자가 아니라 입력한 길이만큼 점으로 보여 줍니다.

### 화면 구성

흰색과 연한 회색을 기본으로 사용하고, 빨간색과 주황색을 포인트 색상으로 적용했습니다. 헤더는 화면 위에 고정했고, 회원가입 Form과 결과표는 카드 형태로 구성했습니다.

## 수업 범위 외에 추가한 부분

### 회원가입 도우미

회원가입 화면 오른쪽에 간단한 입력 도우미를 추가했습니다. 빠른 질문 버튼을 누르거나 직접 질문을 입력하면 아이디, 비밀번호, 이메일, 전화번호, 필수 항목, 관심 분야, 약관 동의에 관한 안내를 보여 줍니다.

이 기능은 실제 AI 서비스나 외부 API를 사용하는 방식이 아닙니다. 질문 안에 특정 단어가 포함되어 있는지 확인하고, `signupHelper.js`에 미리 작성한 답변을 보여 주는 규칙형 방식입니다.

채팅용 `form`은 회원가입 `form` 바깥에 따로 두었습니다. 두 Form을 중첩하면 제출 동작이 서로 영향을 줄 수 있기 때문입니다.

---

## 작동 방식

1. `signup.html`에서 회원 정보를 입력합니다.
2. 회원가입 버튼을 누르면 `required`, `pattern`, `minlength` 조건을 확인합니다.
3. 조건이 맞지 않으면 브라우저가 해당 입력칸을 안내하고 제출을 막습니다.
4. 조건을 모두 만족하면 GET 방식으로 `result.html`에 값이 전달됩니다.
5. `result.js`가 `URLSearchParams`로 값을 읽습니다.
6. 관심 분야는 `getAll()`로 선택한 값을 모두 가져옵니다.
7. 결과표에 회원 정보를 표시하고, 다시 입력하기 버튼으로 입력 화면에 돌아갈 수 있습니다.

## 폴더 구성

```text
01_Signup_Result_Page/
├── signup.html
├── result.html
├── signup.css
├── validation.js
├── result.js
├── signupHelper.js
└── media/
    └── skala-logo.jpg
```

| 파일 | 역할 |
|---|---|
| `signup.html` | 회원가입 입력 화면과 도우미의 HTML 구조 |
| `signup.css` | 입력 화면, 결과표, 도우미, 반응형 디자인 |
| `validation.js` | 생년월일 최대 날짜를 현재 날짜로 설정 |
| `result.html` | 전달된 회원 정보를 보여 주는 결과 화면 |
| `result.js` | GET 값 읽기, 복수 선택 처리, 결과표 출력 |
| `signupHelper.js` | 질문의 키워드에 따라 도움말 표시 |
| `media/` | SKALA 로고 이미지 보관 |

## 확인한 내용

- 빈 필수 항목이 있을 때 제출이 막히는지 확인
- 아이디와 비밀번호의 최소 길이가 적용되는지 확인
- 전화번호 형식이 맞지 않을 때 제출이 막히는지 확인
- 정상 입력 시 `result.html`로 이동하는지 확인
- 관심 분야를 여러 개 선택했을 때 모두 표시되는지 확인
- 다시 입력하기 버튼과 회원가입 도우미가 작동하는지 확인
- 작은 화면에서 배치가 한 열로 변경되는지 확인

## 참고 사항

이 프로젝트는 수업 과제로 만든 화면이며 실제 회원 정보를 서버나 데이터베이스에 저장하지 않습니다.  
GET 방식으로 전달된 값은 주소에 표시되므로 실제 서비스에서는 비밀번호와 같은 개인정보를 이런 방식으로 전송하면 안 됩니다.
