# 종합실습 ② 인터랙티브 할 일 관리 앱

## 폴더 구조

```text
02_Interactive_Todo_App/
├── index.html
├── css/
│   └── style.css
├── js/
│   ├── storage.js
│   └── app.js
└── data/
    └── quotes.json
```

## 실행 방법

ES 모듈과 `fetch()`를 사용하므로 `index.html`을 파일 탐색기에서 더블클릭하지 않습니다.

### VS Code

1. `02_Interactive_Todo_App` 폴더 전체를 VS Code에서 엽니다.
2. `index.html`을 마우스 오른쪽 버튼으로 클릭합니다.
3. **Open with Live Server**를 선택합니다.

### 터미널

```bash
python3 -m http.server 5500
```

브라우저에서 `http://localhost:5500`으로 접속합니다.

## 오늘의 한마디 데이터 출처

`data/quotes.json`에 영어 원문 + 자연스러운 한국어 번역 + 저자를 함께 담아두고
`fetch()` + `async/await` + `try/catch`로 비동기 로드합니다. 외부 랜덤 명언 API는
명언 종류가 매우 많아 모든 항목에 자연스러운 번역을 안정적으로 붙이기 어렵기
때문에, 번역까지 포함한 데이터를 프로젝트 안에 두는 방식을 선택했습니다.
`data/quotes.json`을 찾지 못하거나 형식이 잘못된 경우 `js/app.js`의
`DEFAULT_QUOTE_EN` / `DEFAULT_QUOTE_KO` 기본 문구가 표시됩니다.

## [추후 확장 아이디어]

이번 과제의 데이터 모델은 `{ id, text, done }`으로 고정되어 있어 아래 기능은
구현하지 않았습니다. 채점 조건과 데이터 구조를 바꾸지 않기 위한 의도적인 보류이며,
추후 별도 과제나 개인 프로젝트로 확장한다면 다음을 고려할 수 있습니다.

- 날짜별 계획을 볼 수 있는 달력 뷰
- 할 일별 계획 시간 입력
- 할 일별 상세 메모
- 중요도 또는 우선순위 태그
- 마감일 알림 (예: 브라우저 알림, 색상 강조)

## 사용자가 직접 해야 하는 일

1. Live Server 또는 `python3 -m http.server`로 실행합니다.
2. 추가 버튼과 Enter 키를 각각 시험합니다.
3. 빈 값·공백만 입력했을 때 추가되지 않는지 확인합니다.
4. 체크박스, 취소선, 완료 개수·남은 개수·완료율을 확인합니다.
5. 필터 3개와 완료 항목 모두 삭제 버튼을 확인합니다.
6. 새로고침 후 데이터가 유지되는지 확인합니다.
7. "다른 명언 보기" 버튼과, 개발자 도구 Network에서 오프라인으로 전환했을 때
   기본 문구(영문+한국어)가 표시되는지 확인합니다.

코드에서 선택적으로 수정 가능한 문구는 `[사용자 직접 입력]` 주석으로 표시했습니다.
