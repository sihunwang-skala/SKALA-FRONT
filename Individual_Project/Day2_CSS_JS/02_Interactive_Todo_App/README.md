# Interactive Todo App

> 수업 시간에 배운 JavaScript의 배열·객체, DOM, 이벤트, 비동기 처리와 모듈을 한 화면에 연결해 만든 할 일 관리 앱입니다.  
> 할 일을 시간과 함께 등록하고 완료·삭제·필터링할 수 있으며, 데이터는 새로고침 후에도 유지됩니다.

---

## 실행 방법

이 프로젝트는 ES 모듈과 `fetch()`를 사용하므로 `index.html`을 파일 탐색기에서 더블클릭하지 않습니다.

### Live Server

1. `02_Interactive_Todo_App` 폴더 전체를 VS Code에서 엽니다.
2. `index.html`을 마우스 오른쪽 버튼으로 클릭합니다.
3. **Open with Live Server**를 선택합니다.

### 터미널 서버

```bash
python3 -m http.server 5500
```

브라우저에서 아래 주소로 접속합니다.

```text
http://localhost:5500
```

GitHub 저장소:

```text
https://github.com/sihunwang-skala/SKALA-FRONT
```

## 주요 기능

- 추가 버튼 또는 Enter 키로 할 일 등록
- 빈 값과 공백 입력 방지
- 계획 시간 입력 및 시간순 자동 정렬
- 체크박스로 완료 상태 변경
- 완료 항목 취소선 표시
- 개별 할 일 삭제
- 전체·진행중·완료 필터
- 전체·완료·남은 할 일 개수 표시
- 완료율 퍼센트와 진행 막대
- 완료 항목 모두 삭제
- 오늘의 목표 저장 및 복원
- 영어 명언과 자연스러운 한국어 번역 표시
- 다른 명언 보기
- `localStorage` 저장 및 새로고침 후 복원
- 화면 크기에 따른 반응형 배치

## 수업 내용을 바탕으로 확장한 부분

### 계획 시간과 타임라인

할 일마다 계획 시간을 입력할 수 있습니다. 시간 값은 `HH:MM` 형식으로 저장하고, 목록을 그리기 전에 시간순으로 정렬합니다. 시간을 비워 둔 항목은 `시간 미정`으로 표시하고 목록 뒤쪽에 배치합니다.

### 오늘의 목표

할 일 목록과 별도로 하루의 목표를 한 줄로 저장할 수 있습니다. 목표는 할 일 배열과 다른 `localStorage` 키를 사용하므로 데이터가 섞이지 않으며, 새로고침 후에도 유지됩니다.

### 오늘의 한마디

`data/quotes.json`에 영어 원문, 한국어 번역, 작성자를 함께 저장하고 `fetch()`로 불러옵니다. 다른 명언 보기 버튼을 누르면 다시 데이터를 선택하며, 같은 문구가 바로 연속해서 나오지 않도록 처리했습니다.

### 완료율과 일괄 삭제

전체와 완료 개수를 이용해 완료율을 계산하고 진행 막대로 표시합니다. 완료 항목이 있을 때만 `완료 항목 모두 삭제` 버튼이 활성화됩니다.

## 작동 방식

1. 페이지가 열리면 `storage.js`에서 저장된 할 일과 목표를 불러옵니다.
2. `app.js`가 현재 날짜와 기본 시간을 표시하고 `render()`로 목록을 그립니다.
3. 사용자가 할 일과 시간을 입력하면 새 객체를 배열에 추가합니다.
4. 배열이 바뀌면 `saveTodos()`로 저장한 뒤 `render()`를 다시 실행합니다.
5. 체크박스와 삭제 버튼은 부모 `ul`의 이벤트 한 개에서 처리합니다.
6. 필터를 선택하면 조건에 맞는 배열을 만든 뒤 시간순으로 정렬해 표시합니다.
7. 명언 데이터는 `async/await`와 `fetch()`로 불러오며, 실패하면 기본 문구를 표시합니다.

```text
02_Interactive_Todo_App/
```

|---|---|
| `index.html` | 목표, 명언, 할 일 입력·필터·목록·요약의 HTML 구조 |

| `data/quotes.json` | 영어 명언, 한국어 번역, 작성자 데이터 |

## 확인한 내용

- 추가 버튼과 Enter 키가 모두 작동하는지 확인
- 빈 값과 공백만 입력했을 때 추가되지 않는지 확인
- 한글 입력 후 Enter를 눌렀을 때 마지막 글자가 중복되지 않는지 확인
- 체크박스와 취소선, 개별 삭제가 작동하는지 확인
- 전체·진행중·완료 필터가 맞게 표시되는지 확인
- 계획 시간이 빠른 순서대로 정렬되는지 확인
- 전체·완료·남은 개수와 완료율이 맞게 계산되는지 확인
- 완료 항목 모두 삭제 버튼이 필요한 경우에만 활성화되는지 확인
- 새로고침 후 할 일과 오늘의 목표가 유지되는지 확인
- 다른 명언 보기와 명언 로드 실패 시 기본 문구를 확인
- 작은 화면에서 입력창과 버튼 배치가 자연스럽게 바뀌는지 확인

## 구현하면서 확인한 점

한글 입력에서 `keydown`과 `submit`을 동시에 사용하면 마지막 글자가 별도 항목으로 추가되는 문제가 있었습니다. Enter 처리용 이벤트를 따로 두지 않고 `form`의 `submit` 이벤트 하나로 통일해 해결했습니다.

또 예전에 저장된 할 일에는 `time` 값이 없을 수 있으므로, 데이터를 불러올 때 기본 필드의 자료형을 확인하고 시간 값이 없으면 빈 문자열로 보정합니다. 저장 데이터가 손상되었을 때에는 빈 배열을 반환해 앱이 멈추지 않도록 했습니다.

## 참고 사항

- `file://` 방식으로 실행하면 ES 모듈 또는 `fetch()` 오류가 발생할 수 있습니다.
- 이 앱의 데이터는 현재 브라우저의 `localStorage`에만 저장됩니다.
- 다른 브라우저나 다른 기기와 자동으로 동기화되지는 않습니다.
